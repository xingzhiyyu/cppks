# 星轨快线智能购票系统项目说明

课程设计题目：模拟汽车客运公司售票系统  
实现形式：Qt 6 Widgets + C++ + CMake  
项目目录：课程设计工程根目录  
说明用途：作业提交、答辩讲解、现场修改速查  

## 1. 项目整体说明

本程序实现的是一个图形化售票系统。虽然界面文字使用了“车次、列车、席别”等表达，但核心功能和题目要求中的客运售票系统是一致的：可以查询班次、登记乘客身份信息、选择座位、生成订单、查看电子票、退票、改签，并能输出指定班次的旅客登记表。

程序采用 Qt Widgets 编写。界面主体由 `mainwindow.ui` 描述，业务逻辑写在 `mainwindow.cpp`，数据结构和函数声明写在 `mainwindow.h`。项目用 CMake 构建，入口函数在 `main.cpp`。

从用户流程看，程序分为 3 个主页面：

| 页面 | Qt 对象 | 作用 |
|---|---|---|
| 查询页面 | `pageSearch` | 输入出发地、目的地、日期、车型，查询可售班次 |
| 选座页面 | `pageBooking` | 添加乘客，选择席别、车厢和座位，确认支付 |
| 票据页面 | `pageTicket` / `pageAccount` | 查看订单，办理退票、改签，查看账号中心 |

页面切换由 `QStackedWidget` 完成。程序并没有打开多个窗口，而是在同一个主窗口里切换不同页面，这样状态更容易维护。

## 2. 文件组织形式

项目中最重要的文件如下：

| 文件 | 内容 |
|---|---|
| `main.cpp` | 创建 `QApplication`，显示 `MainWindow` |
| `mainwindow.ui` | Qt Designer 生成的界面布局文件 |
| `mainwindow.h` | 数据结构、成员变量、槽函数和辅助函数声明 |
| `mainwindow.cpp` | 业务逻辑、信号槽连接、表格渲染、样式表 |
| `CMakeLists.txt` | Qt 6 工程构建配置 |

### 2.1 程序入口

`main.cpp` 很短，核心代码是：

```cpp
QApplication a(argc, argv);
MainWindow w;
w.show();
return QCoreApplication::exec();
```

含义如下：

- `QApplication` 是 Qt Widgets 程序必须有的应用对象。
- `MainWindow w;` 创建主窗口。
- `w.show();` 显示窗口。
- `exec()` 进入事件循环。按钮点击、表格点击、定时器刷新都依赖这个事件循环。

### 2.2 `.ui` 和 C++ 的分工

本项目没有把所有界面都手写在 C++ 里，而是使用 `mainwindow.ui` 保存静态布局。这样有两个好处：

- 窗口、按钮、输入框、表格等控件位置更直观。
- C++ 代码主要负责行为，不需要堆大量布局代码。

在 `MainWindow` 构造函数中：

```cpp
ui->setupUi(this);
```

这一句会读取 `mainwindow.ui`，创建所有界面控件，并把它们挂到 `ui` 指针下面。例如 `.ui` 里有 `trainTable`，C++ 里就可以写：

```cpp
ui->trainTable->setRowCount(trains.size());
```

因此可以把项目理解为：

| 层次 | 负责内容 |
|---|---|
| `.ui` | 控件有哪些、放在哪里、页面结构是什么 |
| `setupUiLayout()` | 控件怎么连接信号槽、表格怎么初始化 |
| 业务函数 | 查询、购票、选座、退票、改签等实际行为 |
| `getStylesheet()` | 界面颜色、按钮、表格、日期控件样式 |

## 3. 数据结构设计

项目没有使用数据库，所有数据保存在内存容器中。这样适合课程设计演示，逻辑清楚，运行后即可操作。

### 3.1 班次结构 `Train`

```cpp
struct Train {
    QString id;
    QString from;
    QString to;
    QString depTime;
    QString arrTime;
    QString duration;
    int priceBusiness;
    int priceFirst;
    int priceSecond;
    int seatsBusiness;
    int seatsFirst;
    int seatsSecond;
};
```

这个结构保存一个班次的基本信息：

- `id`：班次号，例如 `G101`。
- `from` / `to`：出发站和终点站。
- `depTime` / `arrTime`：出发和到达时间。
- `duration`：历时。
- `priceBusiness`、`priceFirst`、`priceSecond`：不同席别价格。
- `seatsBusiness`、`seatsFirst`、`seatsSecond`：初始余票显示用的数据。

题目中的“班次号、车型、发车时间、终点、座位数量、票价”等要求，主要由这个结构承载。

### 3.2 乘客结构 `Passenger`

```cpp
struct Passenger {
    QString name;
    QString idCard;
    QString seatClass;
    QString seatNo;
};
```

乘客结构用于购票过程中临时保存乘客信息。它对应题目要求中的身份证号、座位号、席别等登记内容。

### 3.3 账号结构 `UserAccount`

```cpp
struct UserAccount {
    QString username;
    QString password;
};
```

账号系统用于区分不同用户的订单。登录后才能出票、退票和改签。这里密码是明文保存，因为课程设计重点是业务流程，不是安全系统。如果答辩时被问到，可以说明真实系统应使用加盐哈希保存密码。

### 3.4 订单结构 `Order`

```cpp
struct Order {
    QString code;
    QString username;
    QString passengerName;
    QString idCard;
    QString trainId;
    QString from;
    QString to;
    QString date;
    QString depTime;
    QString arrTime;
    QString seatClass;
    QString seatNo;
    QString status;
    int fare;
    int refundFee;
    int changeFee;
};
```

订单是出票后的正式记录。它保存乘客、班次、日期、座位、票价、状态、退票手续费和改签手续费。退票和改签不是直接删除订单，而是修改订单状态，这样账号中心仍然可以看到历史记录。

### 3.5 主要成员变量

```cpp
QList<Train> allTrains;
QList<Train> filteredTrains;
Train selectedTrain;
QList<Passenger> passengers;
QSet<QString> selectedSeats;
QMap<QString, UserAccount> accounts;
QList<Order> orders;
QMap<QString, QSet<QString>> soldSeatsByRun;
QString currentUsername;
int activeStep;
```

这些成员变量构成程序的运行状态：

| 变量 | 作用 |
|---|---|
| `allTrains` | 保存所有模拟班次 |
| `filteredTrains` | 保存查询后的班次 |
| `selectedTrain` | 当前正在购买的班次 |
| `passengers` | 当前订单中添加的乘客 |
| `selectedSeats` | 当前订单中已选择的座位 |
| `accounts` | 用户名到账号信息的映射 |
| `orders` | 所有订单记录 |
| `soldSeatsByRun` | 每个日期、班次下已经售出的座位 |
| `currentUsername` | 当前登录用户 |
| `activeStep` | 顶部进度条当前阶段 |

其中最关键的是 `soldSeatsByRun`。它使用 `日期|班次号` 作为 key，例如：

```cpp
QString MainWindow::runKey(const QString &date, const QString &trainId) const
{
    return date + "|" + trainId;
}
```

这样同一个座位在不同日期或不同班次中互不影响，能避免重复售票。

## 4. 程序初始化流程

`MainWindow` 构造函数完成初始化：

```cpp
ui->setupUi(this);
```

执行顺序可以解释为：

4. 调用 `setupUiLayout()` 连接按钮、初始化表格。
5. 调用 `getStylesheet()` 应用 QSS 样式。
6. 默认显示查询页面。

本机时间显示也在构造函数中完成：

```cpp
auto *clockTimer = new QTimer(this);
connect(clockTimer, &QTimer::timeout, this, [this]() {
    ui->timeLabel->setText(QDateTime::currentDateTime().toString("yyyy-MM-dd HH:mm:ss"));
});
clockTimer->start(1000);
```

这里用 `QTimer` 每秒触发一次，把 `timeLabel` 更新为本机当前时间。

## 5. 信号槽机制

Qt 的核心交互机制是信号槽。按钮点击、表格点击、下拉框变化，都会发出信号，然后执行对应槽函数。

例如搜索按钮：

```cpp
connect(ui->searchBtn, &QPushButton::clicked, this, &MainWindow::onSearchClicked);
```

意思是：当 `searchBtn` 被点击时，执行 `onSearchClicked()`。

例如表格双击：

```cpp
connect(ui->trainTable, &QTableWidget::cellDoubleClicked, this, [this](int, int) {
    onBookSelectedTrain();
});
```

这里使用了 lambda 表达式。`[this]` 表示 lambda 内部可以访问当前窗口对象和 `ui`。`(int, int)` 是双击单元格时传入的行列参数，但这里不需要使用，所以没有写变量名。

信号槽是答辩时非常容易被问到的点，可以这样回答：

> Qt 程序不是从上到下执行完就结束，而是进入事件循环。用户点击按钮后，控件发出信号，`connect` 把信号和槽函数绑定起来，所以对应业务函数会被调用。

## 6. 查询班次功能

查询逻辑在 `onSearchClicked()` 中：

```cpp
QString from = ui->fromInput->text().trimmed();
QString to = ui->toInput->text().trimmed();
int typeIdx = ui->trainTypeCombo->currentIndex();
```

先从输入框读取出发地、目的地和车型筛选条件。`trimmed()` 用于去掉首尾空格。

然后遍历全部班次：

```cpp
for (const auto &train : allTrains) {
    if (!train.from.contains(from) || !train.to.contains(to)) {
        continue;
    }
    ...
    filteredTrains.append(train);
}
```

`continue` 表示当前班次不符合条件，跳过它。符合条件的班次加入 `filteredTrains`。

车型筛选通过班次号开头判断：

```cpp
if (typeIdx == 1) {
    if (!train.id.startsWith('G') && !train.id.startsWith('D')) continue;
} else if (typeIdx == 2) {
    if (train.id.startsWith('G') || train.id.startsWith('D')) continue;
}
```

这里约定 `G` / `D` 开头表示高速动车，其他如 `K` / `Z` 表示普通车。

最后调用：

```cpp
renderTrainList(filteredTrains);
```

把查询结果渲染到 `trainTable`。

## 7. 表格渲染方式

项目里大量使用 `QTableWidget`。例如查询结果表格由 `renderTrainList()` 填充。

典型写法是：

```cpp
ui->trainTable->setRowCount(trains.size());
ui->trainTable->setItem(row, 0, readonlyItem(train.id));
ui->trainTable->setItem(row, 1, readonlyItem(QString("%1\n%2").arg(train.depTime, train.from)));
```

含义如下：

- `setRowCount()` 设置行数。
- `setItem(row, col, item)` 设置某个单元格。
- `QString("%1\n%2").arg(...)` 用字符串占位符拼接两行文本。

为了避免用户直接编辑表格，项目写了辅助函数：

```cpp
static QTableWidgetItem *readonlyItem(const QString &text)
{
    auto *item = new QTableWidgetItem(text);
    item->setFlags(item->flags() & ~Qt::ItemIsEditable);
    return item;
}
```

`~Qt::ItemIsEditable` 的意思是从原本的 flags 中去掉“可编辑”标记。这样表格只作为展示和选择，不作为自由编辑输入。

## 8. 购票和选座流程

用户选择班次后，会调用 `onBookSelectedTrain()`，再进入 `onBookTrain(trainId)`。

### 8.1 进入选座页面

`onBookTrain()` 根据班次号找到对应 `Train`，并清空上一笔订单的临时状态：

```cpp
passengers.clear();
selectedSeats.clear();
ui->passengerTable->setRowCount(0);
ui->newPassengerName->clear();
ui->newPassengerId->clear();
```

之后根据班次类型加载席别：

```cpp
if (isHighSpeed) {
    if (selectedTrain.priceSecond > 0) ui->newPassengerSeatClass->addItem("二等座");
    if (selectedTrain.priceFirst > 0) ui->newPassengerSeatClass->addItem("一等座");
    if (selectedTrain.priceBusiness > 0) ui->newPassengerSeatClass->addItem("商务座");
} else {
    ui->newPassengerSeatClass->addItem("硬座");
    if (selectedTrain.priceSecond > 0) ui->newPassengerSeatClass->addItem("硬卧");
    if (selectedTrain.priceFirst > 0) ui->newPassengerSeatClass->addItem("软卧");
}
```

这段代码的核心思想是：不同车型对应不同席别，价格大于 0 的席别才显示。

### 8.2 添加乘客

添加乘客在 `onAddPassenger()` 中完成。主要校验包括：

- 姓名不能为空。
- 身份证号必须是 18 位。
- 单笔订单最多 5 名乘客。
- 同一订单中身份证号不能重复。

身份证号输入框还使用了正则校验：

```cpp
ui->newPassengerId->setValidator(
    new QRegularExpressionValidator(QRegularExpression("^[0-9xX]{0,18}$"), this)
);
```

这个正则表示：只能输入数字或 `x/X`，最多 18 位。

### 8.3 车厢总览

车厢总览由 `setupCarriageMap()` 创建。高速车的车厢规则是：

| 车厢 | 席别 |
|---|---|
| 01车 | 商务座 |
| 02车、03车 | 一等座 |
| 04车、05车、06车 | 二等座 |

普通车的规则是：

| 车厢 | 席别 |
|---|---|
| 01车 | 软卧 |
| 02车、03车 | 硬卧 |
| 04车、05车 | 硬座 |

代码使用 `QTableWidgetItem::setData()` 保存隐藏数据：

```cpp
item->setData(Qt::UserRole, carriage.seatClass);
item->setData(Qt::UserRole + 1, carriage.no);
```

`setData()` 可以把额外信息藏在单元格中。界面显示的是“03车 一等座”，但程序还能从单元格取出真正的席别和车厢号。

### 8.4 座位图生成

座位图由 `setupCabinGrid()` 生成。不同席别的座位列由 `seatColumnsForClass()` 决定：

```cpp
if (seatClass == "商务座") {
    return {"A", "过道", "F"};
}
if (seatClass == "一等座") {
    return {"A", "C", "过道", "D", "F"};
}
return {"A", "B", "C", "过道", "D", "F"};
```

这样可以实现：

- 商务座一排 2 个座位。
- 一等座一排 4 个座位。
- 二等座一排 5 个座位。

座位编号使用：

```cpp
QString seatId = QString("%1 %2%3").arg(carriageNo).arg(row).arg(letter);
```

例如 `03车 4A`。这个字符串同时用于显示、订单保存和防止重复售票。

### 8.5 防止重复售座

已售座位保存在：

```cpp
QMap<QString, QSet<QString>> soldSeatsByRun;
```

查询某个日期某个班次的已售座位：

```cpp
const QSet<QString> soldSeats = soldSeatsByRun.value(runKey(date, selectedTrain.id));
```

如果座位已经售出：

```cpp
bool occupied = soldSeats.contains(seatId);
```

座位会显示为不可选。确认支付时还会再检查一次：

```cpp
for (const QString &seatId : selectedSeats) {
    if (soldSeats.contains(seatId)) {
        QMessageBox::warning(this, "座位已售", ...);
        return;
    }
}
```

这样即使界面状态有延迟，也不会卖出重复座位。

### 8.6 自动分配座位

如果乘客还没有全部手动选座，系统会自动分配剩余座位：

```cpp
const QString seatId = autoSeatForRun(date, selectedTrain.id, passengers.value(seatAssigned).seatClass);
```

`autoSeatForRun()` 会从对应席别的默认车厢开始，从第 1 排开始寻找第一个未售出的座位。找不到则返回空字符串，表示无票可售。

## 9. 订单、电子票和账号中心

确认购票在 `onConfirmBooking()` 中完成。核心步骤是：

1. 检查是否登录。
2. 检查是否添加乘客。
3. 如果座位不足，尝试自动分配。
4. 检查座位是否已售。
5. 把座位写入 `soldSeatsByRun`。
6. 为每个乘客生成一个 `Order`。
7. 跳转到电子票页面。

生成订单时：

```cpp
Order order;
order.code = QString("SG-%1-%2").arg(selectedTrain.id, QString::number(orders.size() + 1).rightJustified(4, '0'));
order.username = currentUsername;
order.passengerName = p.name;
order.idCard = p.idCard;
order.trainId = selectedTrain.id;
...
orders.append(order);
```

`rightJustified(4, '0')` 用来生成 4 位订单序号，例如 `0001`。

账号中心和电子票页面复用同一个订单渲染函数：

```cpp
renderOrdersIntoTable(ui->ticketTable, &selectedOrderRow);
renderOrdersIntoTable(ui->accountOrderTable, &selectedAccountOrderRow);
```

函数内部只显示当前登录账号的订单：

```cpp
if (order.username != currentUsername) {
    continue;
}
```

所以不同用户之间不会互相看到订单。

## 10. 退票功能

退票函数是 `onRefundTicket()`。它先通过 `selectedOrderIndex()` 找到当前选中的订单。

手续费计算：

```cpp
order.refundFee = order.fare * (isAfterDeparture(order) ? 50 : 20) / 100;
```

如果当前时间已经晚于发车时间，收 50%。否则收 20%。

时间判断在 `isAfterDeparture()` 中：

```cpp
const QDate date = QDate::fromString(order.date, "yyyy-MM-dd");
const QTime time = QTime::fromString(order.depTime, "HH:mm");
return QDateTime::currentDateTime() >= QDateTime(date, time);
```

退票后还要释放座位：

```cpp
soldSeatsByRun[runKey(order.date, order.trainId)].remove(order.seatNo);
```

这样别人之后可以买到这个座位。

## 11. 改签功能

改签函数是 `onChangeTicket()`。它不是使用下拉框，而是弹出一个 `QDialog`，里面放一个 `QTableWidget` 显示可改签班次。

候选班次筛选条件：

```cpp
if (train.id != oldOrder.trainId
    && train.to == oldOrder.to
    && trainHasTickets(train)
    && availableCount > 0) {
    ...
}
```

含义是：

- 不能改签到原来的班次。
- 目的地必须一致。
- 班次有票。
- 当前模拟余座大于 0。

改签成功后会执行：

```cpp
soldSeatsByRun[runKey(order.date, order.trainId)].remove(order.seatNo);
soldSeatsByRun[runKey(newDate, train.id)].insert(newSeat);
```

这两句分别释放旧座位、占用新座位。

改签手续费：

```cpp
order.changeFee = order.fare * (isAfterDeparture(order) ? 20 : 0) / 100;
```

发车前改签免费，发车后改签收 20%。

## 12. 旅客登记表输出

题目要求可以输出指定班次的旅客登记表。项目中由 `onExportManifest()` 实现。

用户输入班次号后，程序遍历所有订单：

```cpp
for (const auto &order : orders) {
    if (order.date == date && order.trainId == trainId && !order.status.startsWith("已退票")) {
        ...
        totalFare += order.fare;
        ++count;
    }
}
```

它只统计：

- 指定日期。
- 指定班次。
- 没有退票的订单。

最后用 `QMessageBox::information()` 显示旅客登记表，并给出票款合计。

## 13. 界面样式 QSS

Qt Widgets 的样式可以通过 QSS 控制，语法和 CSS 类似。项目中 `getStylesheet()` 返回一个很长的字符串：

```cpp
QString MainWindow::getStylesheet() {
    return R"(
        QMainWindow {
            background-color: #f3f4f6;
        }
        QPushButton {
            background-color: #4f46e5;
            color: #ffffff;
            border-radius: 8px;
        }
    )";
}
```

`R"( ... )"` 是 C++ 原始字符串。它的好处是里面可以直接写多行文本，不需要给每个引号和换行转义。

常见 QSS 选择器：

| 写法 | 作用 |
|---|---|
| `QPushButton` | 所有按钮 |
| `QPushButton:hover` | 鼠标悬停按钮 |
| `QPushButton#loginBtn` | 对象名为 `loginBtn` 的按钮 |
| `QTableWidget::item:selected` | 表格被选中单元格 |
| `QDateEdit::drop-down` | 日期控件右侧下拉区域 |
| `QCalendarWidget QSpinBox` | 日历里年份编辑框 |

如果答辩时要现场改按钮颜色，可以在 `getStylesheet()` 中找到：

```css
QPushButton {

```

## 14. 常见 C++ / Qt 语法说明

### 14.1 `QString`

Qt 中常用 `QString` 保存文本，而不是标准库的 `std::string`。它对中文支持更方便，也能直接和 Qt 控件交互。

常用写法：

```cpp
QString text = ui->fromInput->text().trimmed();
QString msg = QString("当前车次: %1").arg(selectedTrain.id);
```

### 14.2 `QList`、`QMap`、`QSet`

| 容器 | 类似概念 | 项目用途 |
|---|---|---|
| `QList<T>` | 数组列表 | 保存班次、乘客、订单 |
| `QMap<K,V>` | 字典 | 用户名到账号，班次日期到已售座位 |
| `QSet<T>` | 集合 | 保存已选座位、已售座位 |

`QSet` 很适合判断座位是否已售，因为 `contains()` 语义清楚：

```cpp
if (soldSeats.contains(seatId)) {
    ...
}
```

### 14.3 `const auto &`

遍历列表时经常写：

```cpp
for (const auto &train : allTrains) {
    ...
}
```

含义是：

- `auto` 让编译器自动推断类型。
- `&` 表示引用，避免复制整个对象。
- `const` 表示循环中不修改这个对象。

### 14.4 三目运算符

项目中有这样的写法：

```cpp
const QString countStr = (count > 9) ? "有票" : (count > 0 ? QString("%1张").arg(count) : "无票");
```

它等价于：

```cpp
QString countStr;
if (count > 9) {
    countStr = "有票";
} else if (count > 0) {
    countStr = QString("%1张").arg(count);
} else {
    countStr = "无票";
}
```

答辩时如果觉得三目运算符不好解释，可以说它是 `if else` 的简写。

### 14.5 `setData()` 和 `data()`

表格单元格显示的文字不一定够用，所以项目用 `setData()` 保存隐藏信息：

```cpp
item->setData(Qt::UserRole, train.id);
```

读取时：

```cpp
const QString trainId = ui->trainTable->item(row, 0)->data(Qt::UserRole).toString();
```

这个技巧适合在界面上显示友好文本，同时保留真实数据。

### 14.6 lambda 表达式

例如：

```cpp
connect(ui->newPassengerSeatClass, &QComboBox::currentTextChanged, this, [this]() {
    selectedSeats.clear();
    updateCarriageSelection(ui->newPassengerSeatClass->currentText());
    setupCabinGrid();
    updateSummary();
});
```

这表示下拉框文字变化时，直接执行一段匿名函数。这里不用单独声明槽函数，适合处理较短的界面联动逻辑。

## 15. 功能要求对照表

| 题目要求 | 项目实现 |
|---|---|
| 输入班次基本信息 | `initMockData()` 中初始化模拟班次 |
| 查询指定日期、目的地班次 | `onSearchClicked()` 根据出发地、目的地、车型筛选 |

| 未售座位中选择座位 | `setupCabinGrid()` 和 `onSeatCellClicked()` |
| 系统自动选择座位 | `autoSeatForRun()` |
| 座位不能重复销售 | `soldSeatsByRun` + 支付前二次检查 |
| 退票 | `onRefundTicket()` |

| 输出旅客登记表和票款合计 | `onExportManifest()` |

需要说明的是，题目中“每天创建后天票源数据”在本项目中用“按日期和班次动态生成座位售卖状态”代替。也就是说，程序不预先生成每一天的全部票源表，而是在某日期某班次第一次售票时，通过 `soldSeatsByRun[date|trainId]` 自动形成该日期班次的售座记录。这种写法更简洁，也符合“第一张票售出时创建登记表”的思路。

### 17.10 现场遇到重定义错误怎么办

如果 Qt Creator 提示 `Redefinition of 'row'` 或类似错误，通常是同一个作用域里重复定义了变量。

例如错误写法：

```cpp
int row = 0;
for (int row = 0; row < 10; ++row) {
    ...
}
```

可以改成不同名字：

```cpp
int selectedRow = 0;
for (int r = 0; r < 10; ++r) {
    ...
}
```

现场修改时，循环变量建议用 `r`、`c` 表示行列，外层选中行用 `selectedRow`，这样不容易冲突。

## 18. 可以诚实说明的不足

答辩时如果老师问“还能怎么改进”，可以从这些点回答：

- 当前数据保存在内存中，程序关闭后订单会消失。真实系统应接入数据库或文件持久化。
- 密码目前是明文保存，真实系统应使用哈希和盐。
- 班次数据是模拟数据，真实系统应由调度后台录入或从文件读取。
- 当前余票数量和座位图使用了简化模型，每个席别只展示 6 排座位，适合演示但不是完整车厢。
- 改签候选目前按同目的地筛选，真实系统还应考虑发车日期、余票同步、价格差额等规则。

这些不足不一定是扣分点，主动说明反而能体现你知道真实系统和课程设计之间的边界。

## 19. 总结

这个程序的核心不是复杂算法，而是把一个售票业务拆成清晰的状态和事件：

- 班次数据保存在 `allTrains`。
- 查询结果保存在 `filteredTrains`。
- 当前购票乘客保存在 `passengers`。
- 已选座位保存在 `selectedSeats`。
- 已售座位保存在 `soldSeatsByRun`。
- 正式订单保存在 `orders`。

用户每次点击按钮都会触发一个槽函数，槽函数修改这些状态，然后刷新表格、座位图或订单列表。只要能讲清楚“数据在哪里、按钮点了调用谁、状态如何更新、界面如何刷新”，整个项目的组织形式就讲清楚了。
